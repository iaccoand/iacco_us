Design

Good design starts with a good design process... one that focuses on the user. Iterations of design and testing early in the process minimizes the need for changes later in the development process. Listed below are examples of techniques employed to develop and verify software requirements before any code is written.

  • UI Sketches - Pencil and paper, markers, whiteboards.
  • Prototypes - Low fidelity and high fidelity prototypes.
  • User Testing - Remote and in-person task based testing.
  • A|B Testing - Automated testing using software and analytics.

User Centered Design and Development Process

Research & Analysis Understand needs of the user, context of use, business requirements, and dev team capabilities.

Lo-Fi Designs Rough sketches, wireframes, or rapid prototyping tools. Prepare designs for review by development team.

Dev Team Review

User Testing Evaluate the designs with actual users. Verify the designs will be successful before moving on.

Hi-Fi Designs Develop the frontend UI, or a realistic prototype. Simulate the finished product and user test again.

Dev Team Review

Requirements & Development Write requirements after consulting with the development team. Develop the software.

QA and Testing Verify the finished product meets the requirements. User test the feature on the test server before pushing to production.
